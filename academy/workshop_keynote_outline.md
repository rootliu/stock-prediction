## Part 2: AI Coding 从哪来 (20 min)

> **核心信息：AI Coding 不是突然冒出来的，而是60年积累的一次爆发**

### 2.1 五个阶段，一条主线

> 📊 展示图表: `diagrams/ai_coding_evolution.drawio`

| 阶段 | 时间 | 一句话总结 |
|------|------|----------|
| 规则与符号 | 1960s-2016 | 手写规则让机器写代码，又慢又脆 |
| Transformer 革命 | 2017-2020 | 注意力机制 + 预训练，统一了范式 |
| 代码专用大模型 | 2021-2022 | Codex 和 Copilot 出现，AI 第一次进入程序员的IDE |
| 指令对齐与推理 | 2023-2024 | GPT-4/Claude/DeepSeek，AI 开始"理解"代码而不只是"模仿" |
| Agent 自主编程 | 2025+ | AI 不再等你下指令，它自己规划、执行、调试、交付 |

**讲述要点**：
- 2021年是分水岭：GitHub Copilot 发布，AI 第一次大规模进入日常开发
- 性能飞跃：HumanEval 基准从个位数飙升到 95%+
- 类比帮助理解：从"计算器"到"实习生"到"初级工程师"到"团队"

### 2.2 一个关键类比

> 从操作系统的演进理解 AI Coding：

| 过去 | 现在 |
|------|------|
| 进程：主动执行，但需要人写每一行指令 | Agent：主动执行，**自己决定**下一步做什么 |
| 对象：被动封装，等着被调用 | LLM 助手：被动等你提问 |

**一句话**：我们正在从"人写代码让机器执行"走向"人说需求让 AI 编程"。

## Part 3: AI Coding 是什么 (25 min)

> **核心信息：AI Coding 不是一个工具，而是一个完整的技术生态**

### 3.1 六大领域全景

> 📊 展示图表: `diagrams/ai_coding_taxonomy.drawio`

用 **"金字塔"** 的方式从下往上讲：

**第一层：基础大模型（地基）**

> "所有 AI 编程工具背后，都是大语言模型在驱动"

两类模型：
- 通用模型：GPT-5, Claude Opus 4.6, Gemini, DeepSeek
- 代码专用模型：StarCoder, CodeLlama, Qwen-Coder, DeepSeek-Coder

关键趋势：MoE 架构让 1 万亿参数的模型只需激活 32B，成本骤降。

**第二层：编程助手（效率工具）**

> "这是大多数人今天接触到的层面"

| 产品 | 一句话 |
|------|--------|
| GitHub Copilot | 日处理 1.5 亿条代码建议，最普及的 AI 编程助手 |
| Cursor | 2 年 ARR 超 5 亿美元，AI-first IDE 的代表 |
| TRAE | 字节跳动出品，国内增长最快 |
| 通义灵码 | 阿里云生态，中文编程体验好 |

**第三层：自主 Agent（范式转变）**

> "这是最让人兴奋、也最让外包公司焦虑的层面"

| 助手 | Agent |
|------|-------|
| 你问一句它答一句 | 你说目标它自己干 |
| 补全你正在写的代码 | 自己读代码、改代码、跑测试、修Bug |
| 像查字典 | 像雇了一个初级工程师 |

代表产品：Claude Code, Devin, GPT-5 Codex, Google Jules

**第四层：多 Agent 协作**

> "不是一个 AI 在干活，而是一个 AI 团队"

ChatDev 模拟软件公司（CEO/CTO/程序员角色），MetaGPT 实现角色分工协作。

### 3.2 现场演示 (5 min)

> **建议演示内容**：用 Claude Code 或 Cursor 当场完成一个任务

推荐演示脚本：
1. 打开终端/Cursor
2. 用自然语言说："帮我写一个网页版计算器，要好看，支持加减乘除"
3. 30秒内生成代码
4. 在浏览器中打开展示效果
5. 追加需求："加上历史记录功能"→ AI 自动修改

> **演示目的**：让高管直观感受"说话就能做出程序"的冲击力

## Part 4: AI Coding 改变什么 (20 min)

> **核心信息：AI Coding 正在重新定义"软件工程"这件事**

### 4.1 覆盖软件工程全生命周期

> 📊 展示图表: `diagrams/ai_coding_se_lifecycle.drawio`

**不只是写代码** — AI 已经渗透到每个阶段：

| 阶段 | 过去 | 现在 |
|------|------|------|
| 需求分析 | BA 访谈 + 文档 | AI 模拟用户、自动生成原型 |
| 设计开发 | 程序员逐行编码 | Agent 自主实现、多Agent协作 |
| 测试 | 手写测试用例 | AI 自动生成测试、模糊测试 |
| 代码审查 | 高级工程师逐行 review | PR-Agent、CodeRabbit 自动审查 |
| Bug 修复 | 人工定位 + 人工修复 | SWE-Agent 自动读 Issue、定位、修复、提 PR |
| 部署运维 | 手动 CI/CD 配置 | AI 生成基础设施代码、分析监控告警 |

### 4.2 SWE-bench：最接近真实的评测

> "不是做算法题，而是修真实的 GitHub Bug"

- 500 个来自真实开源项目的 Issue
- AI 需要自己读代码、理解问题、定位文件、写补丁、跑测试
- 最好的 AI（MiniMax M2.5）已经能解决 **80.2%** 的问题

**停顿提问**：
> "想象一下，你的团队接到一个客户的 Bug 报告。现在有一个 AI，它能自己去读代码库、找到问题根因、写好修复代码、还帮你跑完测试。它的成功率已经超过 80%。**这对你们的运维团队意味着什么？**"

### 4.3 对外包公司的六重冲击

| 维度 | 冲击 | 但也是机遇 |
|------|------|-----------|
| **开发效率** | 常规编码效率提升 3-10 倍 | 同样的团队可以接更多项目 |
| **质量保证** | AI 审查比初级工程师更一致 | 降低缺陷率 = 降低返工成本 |
| **人才结构** | 初级编码岗位需求下降 | "AI 编程教练" 新角色出现 |
| **交付模式** | 人天计费模式受挑战 | 转向价值交付、效果计费 |
| **服务范围** | 技术门槛降低 | 可以进入更多垂直行业 |
| **竞争壁垒** | 纯编码能力不再是壁垒 | 领域知识 + AI 工程能力 = 新护城河 |

> **一句话总结**："未来不是 AI 替代程序员，而是 **会用 AI 的团队** 替代不会用 AI 的团队。"

## Part 5: 前沿模型与研究方向 (20 min)

> **核心信息：竞争格局正在被改写，开源模型崛起，成本急剧下降**

### 5.1 最新模型编程能力擂台赛

> "截至2026年3月，谁最能写代码？"

#### SWE-bench Verified 排行（真实Bug修复）

| 排名 | 模型 | 得分 | 参数 | 开源 | 单次成本 |
|------|------|------|------|------|---------|
| 1 | **MiniMax M2.5** | **80.2%** | 229B MoE | Yes | ~$0.30/M |
| 2 | **GLM-5** | **77.8%** | 744B MoE | Yes | 订阅制 |
| 3 | **Kimi K2.5** | **76.8%** | 1T MoE | Yes | $0.60/M |
| 4 | Claude Opus 4.6 | 71.7~81.4% | 未公开 | No | $5/M |
| - | GPT-5/5.4 | 未公开 | 未公开 | No | ~$10/M |

#### Aider 编程排行（225道编程题）

| 排名 | 模型 | 得分 | 成本/次 |
|------|------|------|---------|
| 1 | **GPT-5 (High)** | **88.0%** | $29.08 |
| 2 | o3-pro | 84.9% | $146.00 |
| 3 | Gemini 2.5 Pro | 83.1% | - |
| 4 | Claude Opus 4 (32K) | 72.0% | $65.75 |

**讲述要点**：
- 没有一个模型在所有维度上都是第一
- GPT-5 在函数级编程最强，MiniMax 在项目级Bug修复最强
- 开源模型已经在核心指标上追平甚至超过闭源旗舰
- **成本差异惊人**：MiniMax $0.30 vs Claude $5.00 vs GPT-5 $10.00（每百万token）

### 5.2 三个值得关注的趋势

**趋势一：Agent 能力爆发**
- Kimi K2.5 的 Agent Swarm：100 个子Agent并行，1500步协调
- MiniMax M2.5 内部已有 30% 任务由 Agent 自主完成
- GLM-5 论文标题就叫"From Vibe Coding to Agentic Engineering"

**趋势二：开源 + MoE = 平民化**
- 三大中国模型全部开源、MIT 协议
- MoE 架构：1T 参数只激活 32B，普通服务器可部署
- 对外包公司意义：**可以私有化部署，数据不出公司**

**趋势三：安全问题浮出水面**
- 约 45% 的 AI 生成代码包含已知漏洞模式
- Agent 自主修改代码带来新的安全风险
- 需要"人类在环"的审查机制

### 5.3 六大研究热点（快速过）

| 方向 | 一句话 |
|------|--------|
| RL + 代码推理 | 让 AI 不只是"背答案"，而是真正"想明白" |
| 安全可信 | 45% 的AI代码有漏洞，怎么办？ |
| 扩散模型生成代码 | 非自回归，像画画一样"并行"写代码 |
| 工具链重塑 | 编译器和IDE需要为AI重新设计 |
| 人机协作 | 人和AI之间的控制权怎么流转？ |
| 全生命周期Agent | 从需求到部署，一个AI团队搞定 |

## Part 6: 动手实战 — 分组竞赛 (70 min)

### 6.0 热身 (5 min)

全场统一任务：**让 AI 生成一个 "Hello World" 网页**
- 要求有公司名称、渐变色背景、一段欢迎语
- 第一个在浏览器中成功展示的组 +5 分先发优势
- 目的：熟悉工具、消除"写代码"的心理障碍

### 6.1 规则讲解 & 选赛道 (5 min)

> "接下来一小时，不是我讲，而是你们动手。三个赛道，选一个最让你们团队兴奋的。"

| 赛道 | 风格 | 一句话 |
|------|------|--------|
| **A. 行业洞察报告** | 战略型 | 让AI帮你写一份给董事会的行业分析 |
| **B. 编程挑战** | 创造型 | 零基础用AI做出一个可运行的程序 |
| **C. AI工具评测** | 对抗型 | 当产品经理，横评多个AI编程工具 |

**积分规则**：

| 维度 | 分值 |
|------|------|
| 完成度 | 40 |
| 创意性 | 20 |
| 展示效果 | 20 |
| 速度奖励 | 10 |
| 互评投票 | 10 |

### 6.2 赛道详情

#### 赛道 A：行业洞察报告

选题（三选一）：
- **A1. 印度外包帝国的 AI 危机** — TCS/Infosys 营收趋势、哪些业务最先被替代
- **A2. 中国软件外包的突围之路** — 从人力密集到AI增强的转型路径
- **A3. 2027年的软件公司长什么样** — AI 作为"虚拟员工"的组织架构

操作流程：选题讨论(5min) → 向AI描述需求(10min) → 生成初版(10min) → 迭代优化(15min) → 数据可视化(10min) → 打磨(5min)

Prompt 参考：
```
你是一位资深行业分析师。请帮我撰写一份关于"AI Coding 对印度
软件外包行业冲击"的研究报告。

要求：
- 目标读者：软件公司CEO和董事会
- 篇幅：3000-5000字
- 必须包含：行业数据、AI替代风险分析、机遇分析、对我司的具体建议
- 用数据说话，每个观点都要有支撑
- 生成 2-3 个数据可视化图表的 Python 代码
先给我一个大纲，我确认后再写全文。
```

评分：数据支撑(10) + 逻辑深度(10) + 可执行性(10) + 视觉呈现(10)

#### 赛道 B：编程挑战

项目（三选一）：
- **B1. 贪吃蛇游戏** — 加分：计分板、难度递增、双人模式、公司logo
- **B2. Excel 数据分析工具** — 加分：超支标红、人效图表、PPT输出、趋势预测
- **B3. 公司官网单页** — 加分：动画效果、AI聊天弹窗、中英双语、手机适配

操作流程：选项目(3min) → 描述需求(5min) → 首版实现(10min) → 体验测试(5min) → 迭代改进(15min) → 冲刺加分(10min)

Prompt 参考：
```
请帮我用 HTML + CSS + JavaScript 创建一个网页版贪吃蛇游戏。

要求：
1. 游戏区域 400x400 像素，深色背景
2. 用键盘方向键控制蛇的移动
3. 吃到食物后蛇身变长，分数+1
4. 撞墙或撞自己游戏结束
5. 显示当前分数，有"重新开始"按钮
6. 界面要现代好看，带圆角和渐变色
请把所有代码放在一个 HTML 文件里。
```

评分：可运行(15) + 功能完整(10) + 视觉效果(10) + 加分挑战(每项+2)

#### 赛道 C：AI 工具评测

评测对象（至少选2个对比）：Claude Code / Cursor / TRAE / Copilot / 通义灵码 / ChatGPT

建议测试任务集：

| 任务 | 考察点 |
|------|--------|
| 写函数：输入日期返回星期几 | 基本代码生成 |
| 读CSV画柱状图 | 数据分析+可视化 |
| 给有bug的代码找出并修复 | 调试能力 |
| Python 转 JavaScript | 跨语言理解 |
| 给代码写单元测试 | 软件工程理解 |

操作流程：设计任务(10min) → 制定评分表(5min) → 分头测试(20min) → 填写对比(10min) → 撰写报告(10min)

评分：测试设计(15) + 方法论(10) + 对比深度(10) + 结论价值(5)

## Part 8: 总结 — 外包公司的下一步 (15 min)

### 8.1 回顾开场投票

> "还记得开场那个问题吗？'一年后你们公司有多少人在写代码？' 现在再投一次。"

对比两次投票结果的变化，引发讨论。

### 8.2 三个带走的核心认知

**认知一：AI Coding 已经从"玩具"变成了"生产力"**
- SWE-bench 80%+ 的Bug修复率不是实验室数据，是真实GitHub项目
- Cursor 年收入 5 亿美元说明市场已经在买单

**认知二：竞争格局正在被成本重塑**
- 开源模型 $0.30/M tokens vs 闭源 $10/M tokens — 30倍差距
- 外包公司可以私有化部署，数据不出公司，成本可控

**认知三：最大的壁垒不是技术，是组织转型的速度**
- 工具人人都能买到，差距在于谁先改变工作方式
- "会用 AI 的团队" 替代 "不会用 AI 的团队"

### 8.3 建议的行动计划

| 时间 | 行动 | 预期效果 |
|------|------|---------|
| **本周** | 给每个开发团队配备 AI 编程工具（Cursor/Copilot） | 让团队开始积累使用经验 |
| **1个月内** | 选1-2个低风险项目做 AI 辅助开发试点 | 量化效率提升数据 |
| **3个月内** | 建立"AI 编程规范"和代码审查流程 | 解决安全与质量顾虑 |
| **6个月内** | 培养"AI 工程师"角色，重新定义交付流程 | 形成可复制的AI增强交付模式 |
| **1年内** | 评估开源模型私有化部署，构建自有AI编程平台 | 建立技术壁垒和成本优势 |

### 8.4 结束语

> "今天大家亲手体验了 AI 编程的能力。有人做出了贪吃蛇，有人写出了行业报告，有人评测了多个AI工具。请注意——**这些都是在不到一小时内，由一群'不会写代码'的人完成的。**
>
> 这就是 AI Coding 的今天。而它每个月都在变强。
>
> 问题不是 AI 会不会改变软件行业——**它已经在改变了。** 问题是你们公司，是主动拥抱这个变化的那一批，还是被变化追着跑的那一批。
